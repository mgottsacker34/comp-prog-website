Project Option A: U-Pick-5


Select five of the six assignments below and complete them. They should have a clean and clear user interface. It is a good idea to print the name in bold at the beginning of the program.

  1. Rhombus calculator: Ask the user for four angles and side lengths of a rhombus. Verify that the information they entered makes a valid rhombus. If it does, compute the area of it. If not, inform the user.
  2. Rock-paper-scissors game: Make a rock-paper-scissors game. Ask the user to pick rock, paper, or scissors, and have the computer randomly generate rock, paper, or scissors. Play until the user or the computer wins two games.
  3. Craps simulator: Craps is a relatively simple dice game played with two dice. One of the rules is that if the player rolls a 7 or 11, they win immediately. Simulate the Craps dice roll one million times. Output the percentage of the time that the user wins immediately.
  4. Word searcher: Prompt the user to enter a long string of text. Prompt the user to enter a string for which they want to search in the long string of text. Output the number of times the string occurs.
  5. Password generator: Randomly generate a 12-character password consisting of lowercase letters, uppercase letters, numbers, and symbols. For each character, your program should first randomly choose to which character class the random character will belong. For the symbols, use the range of characters from '!' to '/' (ASCII decimal values of 33 to 47).
  6. Password cracker: Passwords are rarely stored on servers in plaintext. Instead, each user's plaintext password is passed through a hashing algorithm, which produces a string of numbers and letters that is the same length regardless of the length of the user's password. A common hashing algorithm is the MD5 algorithm. Given the string aaa, MD5 produces 47bce5c74f589f4867dbd57e9ca9f808. The resultant hash is fast to compute, but computationally expensive to reverse. This quality makes hashed passwords resistant to hackers (to some degree) because if they obtain the list of hashed passwords, they cannot read them directly and input them into a sign-in page. However, they can employ looping to develop a brute force attack on a hashed password. In this kind of attack, hackers write a program to cycle through every possible combination of characters, compute each combination's hash, and compare it to the hash they are trying to crack.

    Write a password cracker. The program should first randomly generate a 3-character password in the character range '!' to '~' (ASCII decimal values 33 to 126). Compute its hash with the method below. Then, use Java to generate combinations of 3 characters and compute their hashes until it finds a matching hash. Note that this process may take a while. Output the matching hash and plaintext, and the number of tries it took the computer to find the match. Use the following method (note that you will need to import java.security.* and java.math.*): 

    
// Compute MD5 hash string of a plaintext string
public static String computeMD5Hash (String plaintext) {
    String hashtext = "";
    try {
        MessageDigest m = MessageDigest.getInstance("MD5");
        m.reset();
        m.update(plaintext.getBytes());
        byte[] digest = m.digest();
        BigInteger bigInt = new BigInteger(1,digest);
        hashtext = bigInt.toString(16);
        while(hashtext.length() < 32 ){
            hashtext = "0"+hashtext;
        }
    } catch (NoSuchAlgorithmException ae) {
        System.exit(2);
    } 
    return hashtext;
}

    You could also consider using the following method in generating candidate strings: 

    
// Generate a random character between ch1 and ch2 
public static char getRandomCharacter(char ch1, char ch2) {
    return (char)(ch1 + Math.random() * (ch2 - ch1 + 1));
}



Project Option B: Game


Write a text adventure game in Java. This project option is more open-ended, but has some stipulations.

Your game must include all of the following:

  • A title.
  • 5 stages, each with its own task or activity for the user. Each stage must have at least one user interaction (gathering input from user). The code for each stage must be in its own method.
  • A character representing the user. The character must have:
    1. a name (can be user-entered)
    2. some kind of special ability that affects gameplay
  • An instance of ASCII art.
  • An element of luck and three elements of randomness that affect gameplay. This could be a randomly-generated number that affects the user's success overall, or for a specific task.
  • A mathematical computation using a formula more complex than simple math.
  • Some kind of matrix generated with a loop and according to some pattern.
  • Some other looping element.
